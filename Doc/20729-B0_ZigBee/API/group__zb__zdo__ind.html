Detailed Description

The INDICATION interface in ZDO is used to report initiative events to upper layer.


It consists of ZDO specific events such as CONNECTED / DISCONNECTED.
And it also supports to bypass the indication events from lower layer (NWK / APS)

Enumeration Type Documentation

enum zb_zdo_ind_evt

Type of ZDO INDICATION.

Enumerator
ZDO_IND_NODE_CONNECTED 

Node is connected to network.

No param

ZDO_IND_NODE_DISCONNECTED 

Node is disconnected from original network triggered.

No param

ZDO_IND_NODE_LOSS_CONNECT_SYNC_START 

Node loss the network connection and start to try orphan sequence.

No param

ZDO_IND_NWK_NEW_NODE 

Node got updated from the network: New node joined.

This indication will come with param zb_zdo_ind_nwk_node_info_t

ZDO_IND_NWK_CHILD_JOINED 

Node got updated from the network: New child joined.

This indication will come with param zb_zdo_ind_nwk_node_info_t

ZDO_IND_NWK_CHILD_REJOINED 

Node got updated from the network: child rejoined.

This indication will come with param zb_zdo_ind_nwk_node_info_t

ZDO_IND_NWK_CHILD_LEFT 

Node got updated from the network: Child left This indication will come with param zb_zdo_ind_nwk_node_info_t.

ZDO_IND_NLME_IND 

NLME indication primitive w/ param zb_zdo_ind_nlme_apsme_param_t
This indication only visiable when user enable it.

ZDO_IND_APSME_IND 

APSME indication primitive w/ param zb_zdo_ind_nlme_apsme_param_t
This indication only visiable when user enable it.
