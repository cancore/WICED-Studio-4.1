Typedef Documentation

typedef struct _ZBAPP_ZCL_CB_CLST_CMD_INPUT_t ZBAPP_ZCL_CB_CLST_CMD_INPUT_t

Input structure for AO-defined callbacks of CB_CLST_CMD type.

If "pRxDataPriv" is CNF, "pConfirmedTxDataNode" is valid.
If "pRxDataPriv" is IND, "pAoClst" is valid.
If "pRxDataPriv" is RSP, "pAoClst", "pRespondedTxDataNode" and "bDefaultRsp" are valid.

typedef struct _ZBAPP_ZCL_MGMT_CB_INPUT_t ZBAPP_ZCL_MGMT_CB_INPUT_t

Structure for registering and executing AO-defined callbacks.


For registering cb_clst_cmd, specify "aoId", "clstId", "cmdId", "cbType<CB_MODIFY", "cb.clstCmd", and then call "zbapp_zcl_register_cb".
For registering cb_modify, specify "aoId", "clstId", "cbType==CB_MODIFY", "cb.modify", and then call "zbapp_zcl_register_cb".
For registering cb_clst_proc, specify "aoId", "clstId", "cbType==CB_CLST_PROCESS", "cb.clstProc", and then call "zbapp_zcl_register_cb".
For executing cb_clst_cmd, specify "aoId", "clstId", "cmdId", "cbType<CB_MODIFY", and then call "zbapp_zcl_find_and_exec_cb".
For executing cb_modify, specify "aoId", "clstId", "cbType==CB_MODIFY", and then call "zbapp_zcl_find_and_exec_cb".
For executing cb_modify, specify "aoId", "clstId", "cbType==CB_CLST_PROCESS", and then call "zbapp_zcl_find_and_exec_cb".

enum _ZBAPP_ZCL_CB_TYPE

Callback types for AO to register.

Enumerator
CB_CLT_IND 

IND of CLT CMD, on SRV side.

CB_CLT_CNF 

CNF of CLT CMD, on CLT side.

CB_CLT_RSP 

RSP of CLT CMD, on CLT side.

CB_SRV_IND 

IND of SRV CMD, on CLT side.

CB_SRV_CNF 

CNF of SRV CMD, on SRV side.

CB_SRV_RSP 

RSP of SRV CMD, on SRV side.

CB_MODIFY 

Called when attributes change, which includes the following scenario: self-cluster's IND, WRITE_ATTR, scene cluster's IND, dependencies, etc.

CB_CLST_PROCESS 

Called at certain points after receiving IND for special processing.
