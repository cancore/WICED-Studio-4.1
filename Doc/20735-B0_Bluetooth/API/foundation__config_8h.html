Detailed Description

Header for configurable items from the foundation module.

Macro Definition Documentation

#define FOUNDATION_CONFIG_ITEM_ID_COMPRESSED_DATA   0x04

Config item code used to write compressed chunk of data.

Given that this data will be a part of an image, there never needs to be a need to encode such an item from an ENTRY: construct. An encoder subroutine is still provided just in case.

#define FOUNDATION_CONFIG_ITEM_ID_CONFIG_LAYOUT   0x00

Config item code used to communicate information about the rest of the config data.

This includes its layout and properties of the physical media holding it. This item is used in the static section.

#define FOUNDATION_CONFIG_ITEM_ID_CRYSTAL_FREQUENCY_KHZ   0x01

Config item code used for config data to notify firmware of the frequency of the crystal.

This is used so that firmware does not have to (or if firmware cannot) measure the frequency.

#define FOUNDATION_CONFIG_ITEM_ID_DATA   0x03

Config item code used to write an arbitrary chunk of data to an arbitrary address.

The two fields are a 32-bit address (named address in ENTRY constructs), and data with arbitrary length. The length is implied by the size of the array specified for data, as in data = {0x01, 0x02};

#define FOUNDATION_CONFIG_ITEM_ID_DYNAMIC_MEMORY_MIN_ADDRESS   0x016

Config item code for the size of HW AES scratch buffer.

Config item code for setting initial value of minimum address used in permanent memory allocation

#define FOUNDATION_CONFIG_ITEM_ID_FUNCTION_CALL   0x06

Config item code used to call a function directly while processing config data.

It takes a single parameter in such an ENTRY: construct, which is function_addresss.

#define FOUNDATION_CONFIG_ITEM_ID_PMU_FLAGS   0x09

Config item code used for configuration of the PMU flags.

See pmu_file.h "PMU configuration bits" for details

#define FOUNDATION_CONFIG_ITEM_ID_PMU_MINIMUM_SLEEP_TIME_MICROSECONDS   0x08

Config item code used for configuration of the minimum sleep time.

If sleep is allowed by all participants in the sleep decision, but firmware would need to wake up before this amount of time has elapsed, fimware won't bother sleeping.

#define FOUNDATION_CONFIG_ITEM_ID_REGISTER_DLL_SRAM_USE   0x05

Config item code used to register the SRAM used by a DLL, which could be a patch or application DLL loaded from config data.

#define FOUNDATION_CONFIG_ITEM_INIT_BB_REGISTER_FIELDS   0x02

Config item code used to poke hardware register values to optimize system performance.

Please note that care should be taken to assure that the register setting changes are not contradictory to the actions of hardware drivers in firmware. For many registers, a firmware change or patch may be more appropriate, or a poke via config may be ineffective once the driver starts doing its job. Application Engineers are requested to inspect the firmware code with this in mind, or consult with the firmware project lead. The item is structured as triplets of register[x].address, register[x].value, and register[x].mask. No count field needs to be provided, since the encoder script will take care of calculating that. Register indices also need not be provided. If register indices are specified as [x] explicitly, then the encoder script will calculate the indices, allowing register pokes to be collected from a variety of separate agi files, or for new register pokes to be added without having to re-index the whole parameter set. Please refer to FOUNDATION_CONFIG_INIT_BB_REGISTER_FIELDS_t for further documentation on each register poke.
