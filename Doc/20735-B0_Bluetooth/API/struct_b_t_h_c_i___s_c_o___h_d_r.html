Detailed Description

Byte-packed structure representing a SCO or eSCO packet header.

The structure format is dictated by the Bluetooth core specification.

Field Documentation

UINT8 data_len

Length of data for this packet.

The data starts with the byte after data_len. That data of this length starts with the byte after params_len is true on the physical transport link, but not necessarily for internal data. The actual location of data (or it it is present at all) depends on the context in which this header structure is used.
