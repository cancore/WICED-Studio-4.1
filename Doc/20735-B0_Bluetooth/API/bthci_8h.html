#define BTHCI_ACL_BCF_FROM_HOST_ACTIVE_SLAVE_BROADCAST   1

Flag value specifying that the packet is sent to all active slaves.

The flag is obtained from ( (conn_handle_and_flags & BTHCI_ACL_BCF_MASK) >> BTHCI_ACL_BCF_SHIFT ) in BTHCI_ACL_HDR_t received from the host.

#define BTHCI_ACL_BCF_FROM_HOST_PARKED_SLAVE_BROADCAST   2

Flag value specifying that the packet is sent to all slaves, whether active or parked.

The flag is obtained from ( (conn_handle_and_flags & BTHCI_ACL_BCF_MASK) >> BTHCI_ACL_BCF_SHIFT ) in BTHCI_ACL_HDR_t received from the host. Please note that the park feature has, for all intents and purposes, been deprectated.

#define BTHCI_ACL_BCF_MASK   0xc000

Mask to obtain the broadcast flag.

The flag is obtained from ( (conn_handle_and_flags & BTHCI_ACL_BCF_MASK) >> BTHCI_ACL_BCF_SHIFT ) in BTHCI_ACL_HDR_t.

#define BTHCI_ACL_BCF_POINT_TO_POINT   0

Flag value specifying that the packet is point-to-point (not broadcast).

The flag is obtained from ( (conn_handle_and_flags & BTHCI_ACL_BCF_MASK) >> BTHCI_ACL_BCF_SHIFT ) in BTHCI_ACL_HDR_t received from or to be sent to the host.

#define BTHCI_ACL_BCF_SHIFT   14

Shift to obtain the broadcast flag.

The flag is obtained from ( (conn_handle_and_flags & BTHCI_ACL_BCF_MASK) >> BTHCI_ACL_BCF_SHIFT ) in BTHCI_ACL_HDR_t.

#define BTHCI_ACL_BCF_TO_HOST_BROADCAST_RX_IN_PARK   2

Flag value specifying that the packet was received as a slave in park.

This means that that the packet was sent by the peer device's host using BTHCI_ACL_BCF_FROM_HOST_PARKED_SLAVE_BROADCAST. The flag is obtained from ( (conn_handle_and_flags & BTHCI_ACL_BCF_MASK) >> BTHCI_ACL_BCF_SHIFT ) in BTHCI_ACL_HDR_t to be sent to the host. Please note that the park feature has, for all intents and purposes, been deprectated.

#define BTHCI_ACL_BCF_TO_HOST_BROADCAST_RX_NOT_IN_PARK   1

Flag value specifying that the packet was received as a slave not in park.

This means that the packet was sent by the peer device's host using either BTHCI_ACL_BCF_FROM_HOST_ACTIVE_SLAVE_BROADCAST or BTHCI_ACL_BCF_FROM_HOST_PARKED_SLAVE_BROADCAST. The flag is obtained from ( (conn_handle_and_flags & BTHCI_ACL_BCF_MASK) >> BTHCI_ACL_BCF_SHIFT ) in BTHCI_ACL_HDR_t to be sent to the host. Please note that the park feature has, for all intents and purposes, been deprectated.

#define BTHCI_ACL_CONN_HANDLE_MASK   0x0fff

Mask to obtain the connection handle (without flags).

The connection handle is obtained from (conn_handle_and_flags & BTHCI_ACL_CONN_HANDLE_MASK) in BTHCI_ACL_HDR_t.

#define BTHCI_ACL_PBF_AMP_COMPLETE   3

The flag is obtained from ( (conn_handle_and_flags & BTHCI_ACL_PBF_MASK) >> BTHCI_ACL_PBF_SHIFT ) in BTHCI_ACL_HDR_t. This flag is only to be used with an AMP controller, per the Bluetooth core specification.

#define BTHCI_SCO_PSF_CORRECTLY_RECEIVED_DATA   0

Value specifying that the packet contains correctly received data.

#define BTHCI_SCO_PSF_NO_DATA   2

Value specifying that the packet is a placeholder for lost data.

The flag is obtained from ( (conn_handle_and_flags & BTHCI_SCO_PSF_TO_HOST_MASK) >> BTHCI_SCO_PSF_SHIFT ) in BTHCI_SCO_HDR_t to be sent to the host. All payload bytes in the packet should be 0x00, per the Bluetooth core specification.

#define BTHCI_SCO_PSF_PARTIALLY_LOST_DATA   3

Value specifying that not all, but at least some, eSCO data was lost.

The flag is obtained from ( (conn_handle_and_flags & BTHCI_SCO_PSF_MASK) >> BTHCI_SCO_PSF_SHIFT ) in BTHCI_SCO_HDR_t to be sent to the host. The payload data bytes corresponding to the missing (e)SCO packets should be 0x00, per the Bluetooth core specification. Please note that valid data might be 0x00 as well.

#define BTHCI_SCO_PSF_POSSIBLY_INVALID_DATA   1

Value specifying that the packet contains possibly invalid data.

The flag is obtained from ( (conn_handle_and_flags & BTHCI_SCO_PSF_TO_HOST_MASK) >> BTHCI_SCO_PSF_SHIFT ) in BTHCI_SCO_HDR_t to be sent to the host.
