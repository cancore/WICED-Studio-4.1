Detailed Description

Implementation for always-on memory management (custom scatterloading, permanent allocations).

Function Documentation

GIVES void* aon_memory_manager_AllocatePermanent ( UINT32  size_bytes,
BOOL32  word_align 
)

Allocate a buffer in always-on memory permanently.

This can be done only in full-FW mode. Invoking this in micro-BCS mode is undefined (safe only in full-FW mode).

Parameters
size_bytesThe number of bytes requested.
word_alignWhen TRUE, ensures that the returned buffer pointer is word aligned. When FALSE, there is no guarantee that the buffer pointer returned will be aligned.
void aon_memory_manager_ScatterLoad ( void  )

Scatterloads the Always-On memory are with ROM defaults.

This must be called once during early cold-boot. When slim/fast boothing, this function shall not be called or retained data will be lost. This function assumes that the standard library functions memset and memcpy are available for use at the point of invocation (library must have been initialized).
